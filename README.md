# 網頁內容說明
## [https://daisord.github.io/](https://daisord.github.io/)

本網頁用以提供分析與彙整臺灣開放資料中的「公有土地資料」，此資料區分為「土地位置資訊」以及「土地屬性資訊」，細節請查看以下分析內容。使用 2026-02-11 詮釋資料進行分析。

資料來源：[https://data.gov.tw/](https://data.gov.tw/)
* 請使用關鍵字搜尋，找到「公有土地資料(土地屬性資訊)」以及「公有土地資料(土地位置資訊)」，並分別下載 XML 與 KML 檔

## 網頁功能使用說明

### 一、載入資料

本工具需分別載入兩種資料檔案，可依需求單獨或合併使用：

#### 1. 載入 XML（土地屬性資料）— 必要步驟

- 將一或多個 `.xml` 檔案**拖曳**至左側「① 拖曳 XML 檔至此」區塊，或點擊「**選擇 XML 檔案**」按鈕從檔案總管選取。
- 支援**多檔同時選取**，系統會合併所有檔案的資料一起顯示。
- 檔案格式需符合 `<Lands>` 根節點結構（國有 / 直轄市有 / 縣市有 / 鄉鎮市有）。
- 載入成功後，畫面下方會顯示統計資訊與資料表格。

#### 2. 載入 KML（地籍 Polygon 圖形）— 選用步驟

- 將一或多個 `.kml` 檔案**拖曳**至右側「② 拖曳 KML 檔至此」區塊，或點擊「**選擇 KML 檔案**」按鈕選取。
- 支援**多檔同時選取**，KML 中的每個 Placemark 會依據 `SECTNO`（段代碼）與 `PARCELNO`（地號）自動與 XML 資料關聯。
- KML 可在 XML 載入**前或後**匯入，載入後會自動重新對比並更新表格。

> **注意**：XML 與 KML 的地號格式不同，工具會自動將 XML 的 8 碼地號（如 `01890000`）轉換為 KML 格式（如 `189`）後進行比對，無需手動處理。

## 三、KML 檔案內部結構

每個 KML 檔案對應一個**地段**，內含該地段下多個**地號地塊**。

```
kml
└── Document
    ├── name              ← 段號 (SECTNO，4 碼)
    ├── visibility
    ├── Placemark [*]     ← 每筆 = 一個地號
    │   ├── id            ← 地號（如 189、1061-2、360-340）
    │   ├── name          ← 地號 + "地號"（如 "189地號"）
    │   ├── styleUrl
    │   ├── Polygon
    │   │   └── outerBoundaryIs
    │   │       └── LinearRing
    │   │           └── coordinates   ← 多邊形頂點座標 (lon,lat,0.0)
    │   └── ExtendedData
    │       └── SchemaData
    │           ├── SECTNO   ← 段號
    │           └── PARCELNO ← 地號
    └── Schema (id="sect_land")
        ├── SimpleField: SECTNO  (geString)
        └── SimpleField: PARCELNO (geString)
```

### 地號格式範例

| SECTNO | PARCELNO | 說明 |
|--------|----------|------|
| 0005   | 189      | 整數地號 |
| 0005   | 221      | 整數地號 |
| 1600   | 1061-2   | 分號地號（母號-子號） |
| 3411   | 360-340  | 分號地號 |

## 五、資料特性摘要

1. **空間資料**：每個地號均附帶完整多邊形邊界，可直接匯入 GIS 工具（如 QGIS、Google Earth）進行視覺化。
2. **屬性欄位精簡**：每筆資料僅含 SECTNO（段號）與 PARCELNO（地號）兩個屬性，無面積、所有者等額外欄位。
3. **地號含子號**：部分地號以「母號-子號」格式表示（如 `360-340`），反映土地分割歷史。
4. **無缺號但有跳號**：8 碼命名的後 4 碼流水號並非連續（如 e_16001002、e_16001003 後跳至 e_16001006），代表部分地段無公有土地資料而省略。
5. **資料量統計**：4,807 個地段（KML 檔案），推估涵蓋數萬筆以上的個別地號地塊。

# KML 與 XML 欄位關聯分析

## 關聯鍵值（JOIN Key）

兩份資料集可透過以下兩個欄位進行關聯：

| 資料 | 欄位名 | 格式 | 範例 |
|------|--------|------|------|
| KML | `SECTNO` | 4 碼數字字串 | `0005`、`1600` |
| XML | `<段代碼>` | 4 碼數字字串 | `0005`、`1600` |
| KML | `PARCELNO` | 母號[-子號]，無前導零 | `189`、`1061-2` |
| XML | `<地號>` | 8 碼數字（母號4碼+子號4碼） | `01890000`、`10610002` |

## 地號格式轉換規則

XML 地號為 8 碼固定格式（母號 4 碼 + 子號 4 碼），需轉換才能對應 KML 的 PARCELNO：

| XML `<地號>` | 母號（前4碼） | 子號（後4碼） | KML `PARCELNO` |
|-------------|-------------|-------------|----------------|
| `01890000` | 0189 → **189** | 0000 → 0（整筆） | `189` |
| `02210000` | 0221 → **221** | 0000 → 0（整筆） | `221` |
| `10610002` | 1061 → **1061** | 0002 → **2** | `1061-2` |
| `36034000` | 3603 → **3603** | 4000 → **4000** | `3603-4000` |

**轉換公式（虛擬碼）：**
```
母號 = int(地號[0:4])   # 去除前導零
子號 = int(地號[4:8])   # 去除前導零
if 子號 == 0:
    PARCELNO = str(母號)
else:
    PARCELNO = f"{母號}-{子號}"
```

## 可關聯欄位一覽

| KML 欄位 | XML 欄位 | 關聯性 | 說明 |
|---------|---------|--------|------|
| `SECTNO` | `<段代碼>` | **直接關聯**（相同值） | 4 碼段代碼，完全一致 |
| `PARCELNO` | `<地號>` | **需格式轉換**（見上方規則） | KML 去前導零+加破折號，XML 為8碼填充 |
| （無） | `<縣市>` | 可由段代碼推導 | KML 無此欄位 |
| （無） | `<鄉鎮市區>` | 可由段代碼推導 | KML 無此欄位 |
| `coordinates` | `<登記面積>` | 可驗算 | KML 含多邊形座標可計算面積，XML 直接提供 |

## 關聯後可獲得的完整資訊

將 KML（空間幾何）與 XML（屬性資料）透過 `SECTNO + PARCELNO` 關聯後，每筆地號可同時擁有：

- **空間位置**：地塊多邊形邊界座標（來自 KML）
- **行政資訊**：縣市、鄉鎮市區、段小段名稱（來自 XML）
- **地籍資訊**：登記面積、使用分區、使用地類別（來自 XML）
- **價格資訊**：公告現值、公告地價、申報地價（來自 XML）
- **所有權資訊**：所有權人名稱、類別、持分比例、管理者機關（來自 XML）

## 關聯注意事項

1. **對應比例非 1:1**：一個段代碼在 KML 可能對應多個 KML 檔案（8 碼命名），但在 XML 只有一個檔案，需以**段代碼匹配**而非檔案名稱直接對應。
2. **KML 不含所有權分類**：KML 無法區分國有/直轄市有/鄉鎮市有，需透過 XML 的父節點（`<國有>`、`<直轄市有>` 等）判斷。
3. **部分地號僅在一方存在**：XML 記錄所有公有地號，KML 則只有具備明確邊界的地號，兩者可能有少量差異。
4. **地號格式需標準化**：進行 JOIN 前務必將 XML `<地號>` 轉換為 KML `PARCELNO` 格式，否則無法比對。
